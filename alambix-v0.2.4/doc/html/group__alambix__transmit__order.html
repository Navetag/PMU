Detailed Description

Macros and functions to transmit an order to the bartender.

Transmission of the order to the bartender is based on a POSIX message queue (ALAMBIX_BARTENDER_MQ).

In order to transmit an order, the waiter may follow an algorithm like the following:

// transmit the order to the bartender
char * ordered_drink;
while ((ordered_drink = alambix_get_ordered_drink()) != NULL)
{
// send the ordered drink using the bartender message queue
}

In order to receive an order from the waiter, the bartender may follow an algorithm like the following:

// receive the order from the waiter
do
{
// receive the drink order using the bartender message queue
}
while (alambix_has_ordered_drink());

void alambix_get_ordered_drink ( )

Get the next ordered drink.

Get the next drink ordered by an Alambix client.
This function must be called by an Alambix waiter thread (alambix_waiter_thread) after the waiter has taken the order.

Returns
The next drink that has been ordered by the client, null at the end of the ordered drinks set.

int alambix_has_ordered_drink ( )

Say if there is another drink to order (to the bartender).

This function must be called by an Alambix bartender thread (alambix_bartender_thread).

Returns
1 if there is another drink to order to the bartender, 0 otherwise.
